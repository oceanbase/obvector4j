The package centers on the abstract base class `Sqlizable`, which exposes two methods: `toDB(int, PreparedStatement)` for writing values into a prepared statement parameter and `toString()` for human-readable representation. Concrete leaf classes (`SqlInteger`, `SqlFloat`, `SqlDouble`, `SqlText`, `SqlVector`) each wrap a single primitive or array and implement the binding in terms of the corresponding `PreparedStatement` setter — e.g. `ps.setInt`, `ps.setString`. Vector values are wire-encoded as bracketed comma-separated float strings (e.g. `[0.1, 0.2]`). The factory `SqlizableFactory.build(DataType, ResultSet, String)` maps schema `DataType` enum variants to the appropriate `Sqlizable` subclass when reading from a `ResultSet`, with integer types all collapsing to `SqlInteger`, floating-point types to their matching wrappers, and string/vector/text types going through `SqlText`. Dependency direction is one-way: this model package depends on `com.oceanbase.obvector4j.schema.DataType` but has no reverse dependency, keeping it a pure data-binding layer consumed by higher-level DAO/query code.
