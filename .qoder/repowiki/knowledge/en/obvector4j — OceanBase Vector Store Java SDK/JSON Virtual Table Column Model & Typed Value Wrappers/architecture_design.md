Flat package of value-objects plus a small factory/metadata layer:
- `JsonData` is an abstract base with a `toJson()` method; concrete leaf types `JsonBool`, `JsonInt`, `JsonTimestamp`, `JsonDecimal`, `JsonVarchar` each wrap one SQL type.
- `JsonDataType` enum drives the single dispatch in `JsonDataFactory.build(ResultSet, String)`, which reads from JDBC via column name or index and constructs the matching `JsonData` subclass.
- `JsonTableColumn` holds per-column schema (id, name, type string, nullability, default) and owns a `JsonDataFactory` (`jcol_model`) created by parsing the raw `jcol_type` string (TINYINT→BOOL, INT→INT, TIMESTAMP→TIMESTAMP, VARCHAR(n)/VARCHAR→VARCHAR, DECIMAL(p,s)/DECIMAL→DECIMAL) using regex. Its `validation(ResultSet, col_name)` delegates to the factory to materialize `jdata`.
- `JsonTableMetadata` reflects column definitions from the server-side `meta_json_t` table (filtered by `user_id`), caches them in a `HashMap<table, List<JsonTableColumn>>`, parses the stored default-value JSON via `org.json.simple.parser.JSONParser`, and exposes helpers like `checkTableExists` / `checkColumnExists`.
Dependency direction: `JsonTableMetadata` → `JsonTableColumn` → `JsonDataFactory` → `JsonData` subclasses; no cross-package imports except `org.json.simple` for default-value deserialization. The package is explicitly marked internal (`ObVecJsonClient` only).
